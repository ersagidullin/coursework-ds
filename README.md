# coursework-ds

## Правила работы с репозиторием

### 1. Создание Pull Request

* Перед созданием PR необходимо синхронизировать ветку с `main`
* Один Pull Request должен соответствовать **одной задаче**
* В поле **Assignee** указывается тот, кто проводит ревью
* **Merge Pull Request** выполняет Assignee после успешного прохождения проверки

### 2. Именование Pull Request и веток

Название должно соответствовать:

* название задачи на английском языке
* все буквы в нижнем регистре
* слова разделяются символом `_`

**Примеры**

```
add_get_repository_endpoint
update_api_response_format
```

### 3. Работа в отдельных ветках

* Разработка ведется в отдельной ветке
* Ветка создается от актуальной `main`
