# coursework-ds

## Правила работы с репозиторием

### 1. Создание Pull Request

* Pull Request создаётся из **актуальной ветки `main`**
* Перед созданием PR подгрузите обновления из ветки `main`

### 2. Принцип «одна задача — один PR»

* Один Pull Request должен соответствовать **ровно одной задаче**
* Не объединяйте несколько задач в одном PR

### 3. Назначение ответственного

* В поле **Assignee** необходимо указать того, кто будет проводить ревью
* **Merge Pull Request выполняет Assignee** после успешной проверки

### 4. Именование Pull Request

Название PR должно соответствовать следующей схеме:

* название задачи на английском языке
* все буквы в нижнем регистре
* слова разделяются символом `_`

**Пример:**

```
add_get_repository_endpoint
update_api_response_format
```

### 5. Работа с комментариями

* Регулярно отслеживайте комментарии к вашему PR
* После внесения исправлений уведомляйте ревьюера при необходимости
